# CRUD-App
Create Read Update and Delete Functionalities

MVC Structure Notes
Model: 
The database of the webpage.
Holds all the information and data entries that are to be saved
Can use local storage or session storage for this component

View: 
Presentation of the data
Looks for events 
Sends appropriate requests with each event occurrence to the controller
Receives updates from the controller to make changes in the view components

Controller:
Brains of the web application
Handles the manipulation of data according to the requests given by the view, by fetching the appropriate data and functionalities from the Model and sending them back to view
